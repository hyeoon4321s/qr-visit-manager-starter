# 간단 QR 만들기

웹 주소를 입력해 QR 코드를 만들고, 생성한 QR을 같은 화면 아래에 카드 형태로 보관하는 학습용 웹사이트입니다.

## 주요 기능

- 로그인과 관리자 키 없이 QR 코드를 생성합니다.
- Supabase와 데이터베이스를 사용하지 않습니다.
- 생성한 QR 이미지를 PNG 파일로 내려받을 수 있습니다.
- 생성 기록을 현재 브라우저의 `localStorage`에 최대 30개까지 보관합니다.
- 저장된 QR의 주소 복사, PNG 내려받기, 개별 삭제, 전체 삭제를 지원합니다.
- QR에 저장되는 주소는 입력한 원본 웹 주소입니다.

브라우저 저장 방식이므로 다른 컴퓨터나 다른 브라우저와 생성 목록이 공유되지는 않습니다.

## 폴더 구조

```text
qr-visit-manager-starter/
├─ public/
│  ├─ index.html          # QR 생성 화면과 저장 목록
│  ├─ admin.html          # 이전 주소를 홈으로 보내는 안내 화면
│  ├─ css/style.css       # 전체 화면 디자인
│  └─ js/create-qr.js     # 생성, 다운로드, 브라우저 저장 기능
├─ src/server.js          # QR 이미지 생성 API와 정적 파일 서버
├─ package.json           # 실행 명령과 라이브러리 목록
└─ README.md              # 사용 안내
```

## 로컬 실행

```powershell
cd C:\Git\qr-visit-manager-starter
npm install
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

## Vercel 배포

Supabase와 환경변수를 사용하지 않으므로 별도 설정이 필요하지 않습니다.

1. 수정 코드를 GitHub에 푸시합니다.
2. Vercel이 새 커밋을 자동 배포할 때까지 기다립니다.
3. 배포 주소를 열어 웹 주소로 QR을 생성합니다.

이전에 등록한 Supabase 환경변수는 남아 있어도 이 코드에서 사용하지 않습니다.

## 사용 방법

1. QR 이름을 입력합니다. 생략하면 주소의 도메인을 이름으로 사용합니다.
2. `https://` 또는 `http://`로 시작하는 웹 주소를 입력합니다.
3. **QR 코드 만들기**를 누릅니다.
4. 오른쪽에서 생성 결과를 확인하고 PNG 파일로 내려받습니다.
5. 생성한 QR은 화면 아래 **내가 만든 QR 코드** 목록에 자동으로 추가됩니다.

## 저장 방식 안내

생성 목록은 서버가 아닌 현재 브라우저에만 저장됩니다. 브라우저 데이터를 삭제하거나 시크릿 모드를 종료하면 목록이 사라질 수 있습니다. 내려받은 PNG 파일은 영향을 받지 않습니다.
